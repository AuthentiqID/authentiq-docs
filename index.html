Introduction

Hey, if you’re reading this, then you are probably curious how to get started with implementing Authentiq on your website.

We already run Authentiq Connect as a service for you, hence there is no need to install or run anything locally. You can simply get started by including our JavaScript snippet on your page. Dive right in by heading over to the Getting started page.

Of course if you prefer you can install the Authentiq Connect daemon locally as well, now or later. At the moment we have pre-built packages for Debian 7.0, 8.0 and Ubuntu 15.04. You can also install authentiqd from PiPy or build it from source.

Naming

Authentiq Connect
The entire set of the authentication protocol defined by Authentiq, together with the hosted (or installable) server component and the client side JavaScript.
Authentiq Provider
The actual daemon that serves the Authentiq Connect protocol and acts as a bridge between website and app. The provider is also packaged as authentiqd in the context of SysV style init scripts.
